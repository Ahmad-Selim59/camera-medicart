MINGW LINKING FIX - READ THIS IF BUILD FAILS
================================================================================

PROBLEM:
MinGW cannot directly use Visual Studio .lib files. The CameraCtrl.lib was
created with Visual Studio and needs special handling for MinGW.

SOLUTION:
Use build_mingw_simple.bat - it links directly to the DLL file instead of
the .lib file.

================================================================================
IF IT STILL FAILS
================================================================================

Option 1: Generate MinGW import library (Advanced)
---------------------------------------------------
If you have pexports tool:
1. pexports CameraCtrl.dll > CameraCtrl.def
2. dlltool -d CameraCtrl.def -l CameraCtrl.a -D CameraCtrl.dll
3. Then link with: g++ ... CameraCtrl.a ...

Option 2: Use Visual Studio
----------------------------
If you have Visual Studio available, use build.bat instead.

Option 3: Check DLL dependencies
---------------------------------
Make sure CameraCtrl.dll is in the same folder and all its dependencies
are available (usually Windows system DLLs).
